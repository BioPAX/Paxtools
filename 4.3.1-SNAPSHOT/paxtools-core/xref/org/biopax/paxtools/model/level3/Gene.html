4   /**
5    * <b>Definition</b>: A continuant that encodes information that can be inherited through replication.
6    *
7    * <b>Rationale</b>: Gene is an abstract continuant that can be best described as a "schema", a common conception commonly used by biologists to demark a component within genome. In BioPAX, Gene is considered a generalization over eukaryotic and prokaryotic genes and is used only in genetic interactions.  Gene is often confused with DNA and RNA fragments, however, these are considered the physical encoding of a gene.  N.B. Gene expression regulation makes use of DNA and RNA physical entities and not this class.
8    *
9    * <b>Usage</b>: Gene should only be used for describing GeneticInteractions.
10   */
11  public interface Gene extends Entity {
12  
13      /**
14       * An organism, e.g. 'Homo sapiens'. This is the organism that the entity is found in. Pathways may
15       * not have an organism associated with them, for instance, reference pathways from KEGG.
16       * Sequence-based entities (DNA, protein, RNA) may contain an xref to a sequence database that
17       * contains organism information, in which case the information should be consistent with the value
18       * for ORGANISM.
19       *
20       * @return the organism for this gene.
21       */
22      BioSource getOrganism();
23  
24      /**
25       * An organism, e.g. 'Homo sapiens'. This is the organism that the entity is found in. Pathways may
26       * not have an organism associated with them, for instance, reference pathways from KEGG.
27       * Sequence-based entities (DNA, protein, RNA) may contain an xref to a sequence database that
28       * contains organism information, in which case the information should be consistent with the value
29       * for ORGANISM.
30       *
31       * @param source new organism for this gene
32       */
33      void setOrganism(BioSource source);
34  }
