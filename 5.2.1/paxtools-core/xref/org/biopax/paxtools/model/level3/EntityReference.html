7   /**
8    * Definition: An entity reference is a grouping of several physical entities across different
9    * contexts and molecular states, that share common physical properties and often named and treated
10   * as a single entity with multiple states by biologists.
11   *
12   * Rationale:   Many protein, small molecule and gene databases share this point of view, and such a
13   * grouping is an important prerequisite for interoperability with those databases. Biologists would
14   * often group different pools of molecules in different contexts under the same name. For example
15   * cytoplasmic and extracellular calcium have different effects on the cell's behavior, but they are
16   * still called calcium. For DNA, RNA and Proteins the grouping is defined based on a wildtype
17   * sequence, for small molecules it is defined by the chemical structure.
18   *
19   * Usage: Entity references store the information common to a set of molecules in various states
20   * described in the BioPAX document, including database cross-references. For instance, the P53
21   * protein can be phosphorylated in multiple different ways. Each separate P53 protein (pool) in a
22   * phosphorylation state would be represented as a different protein (child of physicalEntity) and
23   * all things common to all P53 proteins, including all possible phosphorylation sites, the sequence
24   * common to all of them and common references to protein databases containing more information
25   * about P53 would be stored in a Entity Reference.
26   *
27   * Comments: This grouping has three semantic implications: <ol> <li>Members of different pools
28   * share many physical and biochemical properties. This includes their chemical structure, sequence,
29   * organism and set of molecules they react with. They will also share a lot of secondary
30   * information such as their names, functional groupings, annotation terms and database identifiers.
31   * <li> A small number of transitions seperates these pools. In other words it is relatively easy
32   * and frequent for a molecule to transform from one physical entity to another that belong to the
33   * same reference entity. For example an extracellular calcium can become cytoplasmic, and p53 can
34   * become phosphorylated. However no calcium virtually becomes sodium, or no p53 becomes mdm2. In
35   * the former it is the sheer energy barrier of a nuclear reaction, in the latter sheer statistical
36   * improbability of synthesizing the same sequence without a template. If one thinks about the
37   * biochemical network as molecules transforming into each other, and remove edges that respond to
38   * transcription, translation, degradation and covalent modification of small molecules, each
39   * remaining component is a reference entity. <li> Some of the pools in the same group can overlap.
40   * p53-p@ser15 can overlap with p53-p@thr18. Most of the experiments in molecular biology will only
41   * check for one state variable, rarely multiple, and never for the all possible combinations. So
42   * almost all statements that refer to the state of the molecule talk about a pool that can overlap
43   * with other pools. However no overlaps is possible between molecules of different groups. </ol>
44   */

46  public interface EntityReference extends UtilityClass, Named, Observable
47  {
48  
49  
50  	/**
51  	 * Variable features that are observed for the entities of this entityReference - such as known PTM
52  	 * or methylation sites and non-covalent bonds. Note that this is an aggregate list of all known
53  	 * features and it does not represent a state itself.
54  	 *
55  	 * @return a set of entityFeatures
56  	 */
57  	@AutoComplete(backward = true)
58  	Set<EntityFeature> getEntityFeature();
59  
60  	/**
61  	 * Variable features that are observed for the entities of this entityReference - such as known PTM
62  	 * or methylation sites and non-covalent bonds. Note that this is an aggregate list of all known
63  	 * features and it does not represent a state itself.
64  	 *
65  	 * @param feature to be added.
66  	 */
67  	void addEntityFeature(EntityFeature feature);
68  
69  	/**
70  	 * Variable features that are observed for the entities of this entityReference - such as known PTM
71  	 * or methylation sites and non-covalent bonds. Note that this is an aggregate list of all known
72  	 * features and it does not represent a state itself.
73  	 *
74  	 * @param feature to be removed.
75  	 */
76  	void removeEntityFeature(EntityFeature feature);
77  
78  
79  	/**
80  	 * Inverse of {@link SimplePhysicalEntity#getEntityReference()}
81  	 *
82  	 * @return a  set of {@link SimplePhysicalEntity} that has this EntityReference
83  	 */
84  	Set<SimplePhysicalEntity> getEntityReferenceOf();
85  
86  
87  	/**
88  	 * @return Controlled vocabulary terms that is used to describe the type of grouping such as
89  	 *         homology or functional group.
90  	 */
91  	Set<EntityReferenceTypeVocabulary> getEntityReferenceType();
92  
93  	/**
94  	 * Adds the given cv to the list of types
95  	 *
96  	 * @param type A controlled vocabulary term that is used to describe the type of grouping such as
97  	 *             homology or functional group.
98  	 */
99  	void addEntityReferenceType(EntityReferenceTypeVocabulary type);
100 
101 	/**
102 	 * Removes the given cv from the list of types
103 	 *
104 	 * @param type A controlled vocabulary term that is used to describe the type of grouping such as
105 	 *             homology or functional group.
106 	 */
107 	void removeEntityReferenceType(EntityReferenceTypeVocabulary type);
108 
109 
110 	/**
111 	 * @return Entity references that qualifies for the definition of this group. For example a member

113 	 */
114 	Set<EntityReference> getMemberEntityReference();  //todo generify?

116 /**
117 	 * Adds the given entityReference to the member list
118 	 *

121 	 */
122 	void addMemberEntityReference(EntityReference entityReference);
123 
124 	/**
125 	 * Removes the given entityReference from the member list
126 	 *
127 	 * @param entityReference An entity reference that qualifies for the definition of this group. For

129 	 */
130 	void removeMemberEntityReference(EntityReference entityReference);
131 
132 	/**
133 	 * Reverse of {@link #getMemberEntityReference()}
134 	 *
135 	 * @return EntityReferences that this EntityReference is a member of.
136 	 */
137 	public Set<EntityReference> getMemberEntityReferenceOf();
