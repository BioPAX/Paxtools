004/**
005 * <b>Definition:</b> A unification xref defines a reference to an entity in an external resource that has the same
006 * biological identity as the referring entity.
007 * <p>
008 * <b>Rationale</b>: Unification xrefs are critically important for data
009 * integration. In the future they may be replaced by direct miriam links and rdf:id based identity management.
010 * <p>
011 * <b>Usage:</b> For example, if one wished to link from a database record, C, describing a chemical compound in a
012 * BioPAX data collection to a record, C', describing the same chemical compound in an external database, one would use
013 * a unification xref since records C and C' describe the same biological identity. Generally, unification xrefs should
014 * be used whenever possible, although there are cases where they might not be useful, such as application to
015 * application data exchange. Identity of interactions can be computed based on the identity of its participants. An
016 * xref in a protein pointing to a gene, e.g. in the LocusLink database17, would not be a unification xref since the two
017 * entities do not have the same biological identity (one is a protein, the other is a gene). Instead, this link should
018 * be a captured as a relationship xref. References to an external controlled vocabulary term within the
019 * OpenControlledVocabulary class should use a unification xref where possible (e.g. GO:0005737).
020 * <p>
021 * <b>Examples:</b> An xref in a protein instance pointing to an entry in the Swiss-Prot database, and an xref in an RNA
022 * instance pointing to the corresponding RNA sequence in the RefSeq database.
023 */
