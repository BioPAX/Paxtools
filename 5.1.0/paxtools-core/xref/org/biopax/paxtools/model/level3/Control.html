5   /**
6    * Definition: An interaction in which one entity regulates, modifies, or otherwise influences a continuant entity,
7    * i.e. pathway or interaction.
8    *
9    * Usage: Conceptually, physical entities are involved in interactions (or events) and the events are controlled or
10   * modified, not the physical entities themselves. For example, a kinase activating a protein is a frequent event in
11   * signaling pathways and is usually represented as an 'activation' arrow from the kinase to the substrate in
12   * signaling diagrams. This is an abstraction, called "Activity Flow" representation,
13   * that can be ambiguous without context. In BioPAX, this information should be captured as the kinase catalyzing
14   * (via an instance of the catalysis class) a Biochemical Reaction in which the substrate is phosphorylated.
15   * Subclasses of control define types specific to the biological process that is being controlled and should be used
16   * instead of the generic "control" class when applicable.
17   *
18   * A control can potentially have multiple controllers. This acts as a logical AND,
19   * i.e. both controllers are needed to regulate the  controlled event. Alternatively multiple controllers can control
20   * the same event and this acts as a logical OR, i.e. any one of them is sufficient to regulate the controlled event.
21   * Using this structure it is possible to describe arbitrary control logic using BioPAX.
22   *
23   * Rationale: Control can be temporally non-atomic, for example a pathway can control another pathway in BioPAX.
24   * Synonyms: regulation, mediation
25   *
26   * Examples: A small molecule that inhibits a pathway by an unknown mechanism.
27   */
